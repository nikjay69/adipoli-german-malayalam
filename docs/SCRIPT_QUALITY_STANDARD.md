# Script Quality Standard — Adipoli German

> Production-grade checklist and anti-patterns for video scripts.
> Use this to evaluate and revise every script before marking ✅ DONE.
> Created: 2026-03-23 (quality pass)

---

## 1. Engagement & Hook (first 60 seconds)

### ✅ DO
- Open with a **concrete scenario** the student will face (exam room, office, restaurant)
- State the **payoff** in one line: "After this video, you can ___."
- Use a **number or stat** to anchor urgency ("12.5% of your exam score")
- Get to the FIRST teaching point within 90 seconds

### ❌ DON'T
- Use vague superlatives: "the MOST IMPORTANT lesson EVER" (every script says this)
- Spend >60s on hype before teaching anything
- Start with "Picture this..." (overused across scripts)

---

## 2. Pacing & Section Balance

### ✅ DO
- Keep sections between **45s–2min** each (target 60–90s)
- **Teach → Drill → Apply** cycle within each concept (not teach-teach-teach then drill)
- Front-load the most useful pattern/phrase — don't save it for minute 5
- Use "micro-drills" (5–10s) between concepts, not just at the end

### ❌ DON'T
- Have monologue blocks longer than 90 seconds without interaction
- Stack 3+ concepts before any practice
- Rush the summary/outro (should be 30–60s of real review, not a bullet dump)

---

## 3. Malayalam Bridge Quality

### ✅ DO
- Use Malayalam when it **genuinely clarifies** (parallel grammar concepts, cultural mappings)
- Keep bridges to 1 sentence — the Malayalam should click instantly or not be there
- Use Manglish (Romanized Malayalam) — no pure Malayalam script in narration
- Best bridges: structural parallels ("nee/ningal = du/Sie"), cultural parallels ("veettu peru = Familienname")

### ❌ DON'T
- Force a Malayalam line into every section — only where it genuinely helps
- Use Malayalam for things that are self-explanatory in English
- Write Malayalam explanations longer than the German point they're supporting
- Use pure translation as a bridge ("'Mein' = ente" is not a bridge, it's a dictionary entry)

---

## 4. Practice Pause Design

### ✅ DO
- Give a **clear prompt** before the pause ("Say: Ich heiße [your name]")
- State the **expected output** format ("Full sentence, not just the word")
- Follow the pause with the **correct answer** + brief reinforcement
- Use graduated difficulty: easy pause → medium pause → challenge pause
- Time the pause to the task: single word = 2s, full sentence = 4s, mini-dialogue = 8s

### ❌ DON'T
- Write "⏸️ Practice!" with no specific prompt
- Use identical pause types throughout (vary: repeat-after-me, fill-blank, free-produce, speed-drill)
- Have more than 8 practice pauses in a 10-minute video (fatigue point)
- Skip the answer reveal after a pause

---

## 5. Exam Usefulness

### ✅ DO
- State the **exact exam section** this maps to (Sprechen Teil 1, Hören Teil 2, etc.)
- Show **exam-format examples** where relevant (actual question types)
- Mark exam strategies with `[EXAM-TIP]` — limit to 3–5 per script
- Connect to the **scoring** ("This gets you X out of 25 points")

### ❌ DON'T
- Vaguely say "this is important for the exam" without specifics
- Put exam tips only in a final section — weave them where they naturally fit
- Overload with exam anxiety — keep it empowering, not pressuring

---

## 6. Scene Structure & Visual Cues

### ✅ DO
- Every visual cue should be **producible**: describe what's on screen specifically
- Use consistent tags: `[SLIDE]`, `[DIAGRAM]`, `[DIALOGUE]`, `[FORM]`, `[TABLE]`
- Include **colour-coding instructions** when grammar elements need visual distinction
- Describe **transitions** between visual types (slide → dialogue → drill screen)

### ❌ DON'T
- Write vague cues: "[SLIDE: something about verbs]"
- Describe visuals that require complex animation without noting it as `[ANIMATION]`
- Forget to specify text that should appear on screen vs. only in narration

---

## 7. Technical Correctness

### ✅ DO
- Verify ALL German sentences are grammatically correct
- Check article-noun agreement (der/die/das + correct Akkusativ/Dativ forms)
- Verify verb conjugation in every example
- Mark any A2+ content with `[A2-PREVIEW]` — brief teasers OK, full teaching not
- Use IPA or clear pronunciation notes for tricky sounds

### ❌ DON'T
- Use simplified/incorrect German "to make it easier" — always model correct German
- Mix up articles (common: das Schnitzel not der Schnitzel)
- Skip gender marking on new nouns — always introduce as der/die/das

---

## 8. Tone & Energy Cues

### ✅ DO
- Mark energy shifts: `[ENERGY: high]`, `[ENERGY: calm]`, `[ENERGY: encouraging]`
- Note where humour lands vs. where to be serious
- Use `[EMPHASIS]` for key phrases the narrator should stress
- Mark `[SLOW]` for German phrases students should hear clearly

### ❌ DON'T
- Write flat throughout — energy should ebb and flow
- Overuse exclamation marks as a substitute for actual energy direction
- Be uniformly hype — calm explanation moments make the energetic moments land harder

---

## 9. Anti-Patterns to Fix in Existing Scripts

| Anti-Pattern | Found In | Fix |
|-------------|----------|-----|
| "the MOST IMPORTANT lesson" hook | v5-1-1, v13-1-1, v2-1-1 | Concrete scenario hook instead |
| Malayalam dictionary entries masquerading as bridges | v5-1-1, v6-3-1 | Cut or convert to structural parallels |
| Long narration blocks (>90s) without interaction | v17-1-1 sections 3-5 | Insert micro-drills mid-section |
| Formulaic "Practice Pause: ⏸️ (N seconds)" | All scripts | Add specific prompts and answer reveals |
| Identical section structure across scripts | All scripts | Vary section types: mini-story, rapid-fire, compare-contrast |
| Summary as bullet dump | v6-3-1, v13-1-1 | Active recall summary ("What were the 3 patterns?") |
| Rushed outro (<15s) | Multiple scripts | Brief but warm close with clear next-step |

---

## 10. Production Readiness Checklist

Before marking a script ✅ DONE:

- [ ] Hook is under 60 seconds and states the concrete payoff
- [ ] No monologue block exceeds 90 seconds
- [ ] Malayalam bridges are genuine (not forced translations)
- [ ] Practice pauses have clear prompts and answer reveals
- [ ] Exam connection is specific (section + scoring)
- [ ] All German is grammatically verified
- [ ] Visual cues are production-ready (specific, tagged)
- [ ] Energy/tone cues are marked at key transitions
- [ ] Teaching follows Teach → Drill → Apply cycle
- [ ] Summary uses active recall, not just repetition
- [ ] Duration estimate is realistic (word count ÷ 130 = minutes of speech)
- [ ] No A2+ content unless marked `[A2-PREVIEW]`
