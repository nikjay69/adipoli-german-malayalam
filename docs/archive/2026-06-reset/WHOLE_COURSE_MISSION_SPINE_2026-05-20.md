# Adipoli German — Whole-Course Mission Spine

Generated: 2026-05-20

## Why this exists

Boss is right: building one beautiful module in isolation is not enough. But trying to redesign all 18 modules at once will create another messy pile.

June 2026 update: this spine now supports the **course/video MVP**, not a games-first rebuild. The spine should help map ~35h of guided A1 lessons, diagnostics, recovery tasks, and Goethe readiness. Module 2 remains a useful first template, but the immediate priority is the whole A1 syllabus + test/recovery system.

The correct path is:

> Define the whole-course spine first, then build one gold-standard vertical slice that proves the spine works.

So Module 2 is not a random module. It is the first **proof-of-direction**. It must be designed with the entire course journey in mind.

## North-star product

Adipoli German should become a course people recommend because it feels:

- made for Malayalis,
- emotionally warmer than generic apps,
- more useful than gamified flashcards,
- more guided than YouTube playlists,
- more exam-serious than casual apps,
- fun enough to return to tomorrow.

The learner should say:

> “This feels like someone built a German course for my actual life.”

## Whole-course product mechanics

### 1. Mission path, not content library

Home should eventually show:

- current mission,
- why it matters,
- progress to today’s win,
- one button to continue.

Other tabs can exist, but they should not dominate first use.

### 2. Fewer game types, deeper quality

Keep only game/exercise patterns that map to learning stages:

- encounter/listen,
- recognize,
- build,
- speak,
- type,
- repair,
- checkpoint.

Delete or quarantine anything that feels like toy clicking.

### 3. Speaking loop is central

Every module needs speaking. Every learner should repeatedly hear:

> “Say it now. Your mouth has to learn German too.”

### 4. Writing loop is central

Every module needs short typed output. Goethe A1 requires writing. Clicking is not enough.

### 5. Mistake repair creates trust

Beginner mistakes should become memorable coaching moments:

- `Ich bin komme` → `Ich komme`
- English letter names while spelling → German letter names
- missing verb position
- wrong formal/informal register

### 6. Kuttan as stakes, not mascot spam

Kuttan should appear when he creates:

- emotional context,
- cultural bridge,
- mistake normalization,
- mission stakes,
- celebration.

Not random chatter.

### 7. Goethe readiness visible but not scary

Readiness should be a meaningful skill map:

- Hören
- Lesen
- Schreiben
- Sprechen

Not just generic progress numbers.

## Vertical slice strategy

The first pilot must include the whole-course DNA:

- mission hook,
- story context,
- audio/listening,
- recognition,
- speaking,
- writing,
- mistake repair,
- celebration,
- next pull,
- clean visual design.

If Module 2 pilot cannot make self-introduction feel great, the larger direction is wrong. If it works, we clone the pattern across the spine.

## What “groundbreaking” means here

Not flashy AI nonsense.

Groundbreaking for this audience means:

- the learner feels seen as a Malayali adult,
- German becomes emotionally approachable,
- every session creates a usable life/exam ability,
- the app feels like a coach guiding them, not a folder of content,
- Kuttan makes them continue without feeling childish,
- the course creates real Goethe confidence.
