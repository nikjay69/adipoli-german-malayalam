# Script Quality Pass Standard (v2.0)

> Reusable standard for upgrading drafted Adipoli German video scripts from "complete" to "production-ready".
> Use this during cron quality passes so revisions stay consistent across modules.
>
> **Created:** 2026-03-23

---

## Goal

A quality pass should make a script:
- less talky
- easier to teach on camera
- easier to follow for Malayalam-speaking beginners
- more exam-useful where relevant
- more accurate and more production-friendly

This is **not** a cosmetic rewrite. It should improve learning performance and shootability.

---

## The Core Upgrade Pattern

Most strong revisions follow this structure:

1. **Hook fast**
   - Get to the payoff in under ~40 seconds
   - Show why this matters in Germany, in the exam, or in real life
   - Avoid long warm-up exposition

2. **Teach in small blocks**
   - Prefer 1 pattern or 3-5 words at a time
   - Do not dump giant tables unless the video is explicitly a reference video
   - Each block should answer: "What can the learner do after this 60-90 seconds?"

3. **Drill immediately**
   - After every concept, include repeat / predict / fill / answer moments
   - Avoid stacking explanations for 2-3 minutes before first learner action

4. **Apply quickly**
   - Move from isolated phrase → mini-sentence → mini-dialogue → exam/life use
   - The learner should feel momentum, not lecture fatigue

5. **Close with an output challenge**
   - Ask for something concrete: 3 sentences, one message, one self-intro, one response, etc.

---

## Preferred Teaching Rhythm

Use this rhythm repeatedly inside a script:

- **Teach** — introduce the pattern
- **Drill** — learner repeats / predicts / answers
- **Apply** — use it in a realistic sentence or dialogue

If a section is running long, split it into two smaller teach-drill-apply cycles.

## Malayalam Bridge Standard

Malayalam should help memory and confidence — not just decorate the script.

### Good Malayalam bridges
Use Malayalam to:
- compare a German pattern with something familiar
- highlight a likely learner confusion
- reassure the learner that something is simpler than it looks
- make a cultural contrast memorable
- add emotional relevance for Malayali migrants/students/workers

### Weak Malayalam bridges
Avoid Malayalam lines that only:
- repeat the English explanation without adding value
- become long side conversations
- break pacing in high-drill sections
- sound funny but do not improve learning

### Rule of thumb
Malayalam should act like a **bridge**, not a detour.

## Exam-Usefulness Standard

When a script has exam relevance, the exam connection must be explicit.

Do not just write "exam relevant" in metadata. Show it in the script.

### Minimum standard
Include at least one of these:
- an exam-style prompt
- a listening discrimination drill
- a 3-point Schreiben reminder
- a Sprechen-safe response template
- a trap warning (time format, dates, spelling, content-point misses, etc.)

### A1 rule
Prefer **simple, score-safe language** over clever language.

If a learner can get the point with two short sentences, do not force a fancy structure.

---

## Dialogue Standard

Golden Dialogues should be active, not passive.

### Prefer
- predict-before-reveal
- pause for response
- fill-the-gap dialogue
- one line at a time with a clear learner task

### Avoid
- long uninterrupted dialogue blocks that learners only listen to
- dialogues that introduce too many new patterns at once

---

## Pacing Standard

A script usually needs tightening if it has any of these problems:
- hook takes more than ~45 seconds to reach the payoff
- too many vocabulary items in one visual block
- explanation paragraphs longer than learner speaking time
- summary repeats everything instead of landing the key tools
- dialogue appears only near the end with no earlier practice

### Usual target improvements
- cut 10-20% of words where possible
- merge repetitive explanation lines
- replace explanation with drill prompts
- reduce 8-item lists to the 4-5 most useful items

---

## Technical Correctness Standard

During quality pass, check:
- German wording and grammar are accurate
- glosses are clear and non-misleading
- article/case usage is appropriate for A1 level
- examples are natural enough for spoken teaching
- exam claims are accurate and not exaggerated
- cultural claims are plausible and not overgeneralized

If simplifying, keep it **correct enough for A1**, not merely catchy.

### Exam-format truth check
For exam-prep scripts, also verify that:
- the named exam part actually matches the real task type
- item counts / timing / scoring claims are consistent with the current course exam map
- Teil 1 / Teil 2 / Teil 3 descriptions are not accidentally swapped or oversimplified
- strategy advice matches the real interaction pattern (match, choose, true/false, form fill, short message, etc.)

A script can feel energetic and still be wrong. **Accuracy beats momentum** for exam-prep videos.

### Speaking-interaction truth check
For **Sprechen** scripts, also verify that the script teaches the exam as an interaction, not as a monologue about the interaction.

Check that:
- Teil 1 scripts favor a short, score-safe self-intro instead of pushing learners into overlong speeches
- likely follow-ups (for example spelling / clarification) are handled if they are central to learner anxiety
- Teil 2 scripts teach both the **question frame** and the likely **short answer / reaction flow**
- Teil 3 scripts teach both the **request frame** and the likely **follow-up response**
- learners get at least a few moments to decide: **question or request?**, instead of only copying model sentences

If the learner can recite patterns but cannot survive a 10-second exchange, the script is still undercooked.

---

## Production-Friendliness Standard

A better script is easier to shoot and edit.

Prefer:
- clean section boundaries
- specific visual suggestions
- compact cards instead of dense tables
- repeatable onscreen patterns
- obvious pause moments for editing and captions

Avoid:
- slides overloaded with text
- sections that need complex visuals without teaching payoff
- narration that depends on dense paragraphs being shown on screen

---

## Quick Revision Checklist

Before marking a script as **REVISED — v2.0**, ask:

- Is the hook concrete and under control?
- Does each section teach one usable thing?
- Are there enough learner speaking / prediction moments?
- Is the Malayalam bridge doing real pedagogical work?
- Is the exam connection explicit where relevant?
- Is the dialogue active rather than passive?
- Did we cut fluff and keep only high-utility content?
- Is the script easier to film now than before?

If the answer is not mostly yes, it is not yet a real quality pass.
