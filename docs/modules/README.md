# Module Planning README

This folder contains the detailed module-by-module planning docs for the full **18-module Adipoli German A1 course**.

These planning docs were built by reviewing:
- `src/lib/content/modules/module-01.ts` through `module-18.ts`
- `docs/COURSE_PLAN_10_10.md`
- existing curriculum and script-planning docs in `docs/`

## What’s in here

Each `MODULE_XX_PLAN.md` file is intended to capture, at minimum:
- **Module title and topic**
- **Teaching Goal**
- **A1 Exam Relevance**
- **Current Content Inventory**
- **Lesson/Video Script Needs**
- **Essential Outcomes**
- **Future Script Direction**
- **Gaps & Blockers**

In practice, many of the stronger launch-plan files also include:
- script status tables
- production exercise gap analysis
- audio/listening requirements
- Malayalam bridge points
- common Malayali learner error patterns
- golden dialogue / template patterns
- concrete next-step checklists

## Quality notes

### Strongest / most operational plans
These generally have the richest “launch plan” treatment:
- Modules **1–14**
- Modules **17–18**

They usually include measurable outcomes, production gaps, script priorities, and next-step implementation notes.

### More lightweight but complete plans
- Modules **15–16**

These are still documented with teaching goals, inventory, script needs, outcomes, future direction, and blockers, but are intentionally lighter because they have lower direct Goethe A1 leverage.

## Recommended production order
Based on `docs/COURSE_PLAN_10_10.md`:
1. **Modules 1–6** — core beginner foundation and high transfer value
2. **Modules 14, 17, 18** — strongest exam leverage
3. **Modules 7–13** — practical survival + grammar extension
4. **Modules 15–16** — culture/bridge content after core exam path is stable

## Key blockers repeated across many modules
- **No real listening layer yet** in most regular lessons
- **No speaking/production exercise coverage** in module data despite strong need
- **Many videos still exist only as outlines or architecture notes**
- **Script coverage is uneven** across modules
- **Audio generation and reusable drill assets** still need systematic rollout
